## BUG 1 — OFFERS: CAN'T SWITCH BETWEEN OFFERS / CAN'T CLICK "CREATE OFFER"

This is a critical UI bug. Diagnose and fix both issues completely.

### Bug 1A: Can't switch/click other offers once one is selected

The offers list/table in `client/src/pages/offers.tsx` has a selection or active-state issue. Once a user clicks on one offer (to view details, edit, or select it), clicking on any other offer does nothing — the UI is stuck on the first clicked offer.

Root causes to investigate and fix:
- Check if there is a `selectedOffer` state that is being set but never cleared or overwritten when a new offer is clicked.
- Check if event propagation is being blocked (`e.stopPropagation()` used incorrectly) preventing click handlers on other rows/cards from firing.
- Check if a Dialog or Sheet is being opened with a stale ref that doesn't update when a different offer is clicked.
- Check if the edit/view dialog is keyed correctly — if `key` is missing on the dialog or is static, React may reuse the component without re-rendering with new data.
- Check if there is a `useEffect` that depends on `selectedOffer` but never re-runs because the dependency comparison fails (e.g., comparing objects by reference).
- Fix: Ensure every offer row/card has an `onClick` that sets `setSelectedOffer(offer)` freshly. Ensure the Dialog/Sheet receiving the offer data has `key={selectedOffer?._id}` so React unmounts and remounts it with fresh state each time. Ensure no stale closures are trapping old offer data.
- After fixing, test: click offer A → details show → click offer B → details update to B → click offer A again → details update back to A. This cycle must work perfectly.

### Bug 1B: "Create Offer" button does nothing when clicked

The "Create Offer" button in `offers.tsx` is unresponsive. Diagnose and fix:
- Check if the button has a missing or broken `onClick` handler.
- Check if `setCreateDialogOpen(true)` (or equivalent state setter) exists and is correctly wired to the button.
- Check if the Create Offer Dialog/Sheet component is rendered in the JSX at all — it may exist in code but not be in the render tree, so opening the state does nothing visible.
- Check if there is a z-index or pointer-events CSS issue making the button visually present but not clickable (e.g., an overlay div sitting on top).
- Check if a loading state or disabled prop is accidentally set to `true` permanently on the button.
- Check if there is a JavaScript error being thrown silently that prevents the click handler from executing — add try/catch around the handler.
- Fix the button so clicking it opens a fully functional Create Offer dialog/sheet with all fields: name, description, offer type, date range, active toggle, item selection with per-item discount values, and a working submit that POSTs to `/api/offers`.
- After fixing, test the full create flow: open dialog → fill fields → add items → submit → offer appears in list → dialog closes → success toast shows.

After fixing both bugs, update code.md under `client/src/pages/offers.tsx` to document the corrected interaction patterns.
